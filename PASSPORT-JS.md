## 5. Проверката

`src/core/checkProblem.js`. Получава ЕДИН сглобен низ HTML.
Изпълнителят се избира по `runtime` в `meta.js` (`js-worker`).

**Типове за код:** `runs`, `returns`, `throws`, `logs`,
`src_contains`, `src_not_contains`, `src_count`, `src_changed`.
Старите имена (`js_runs`, `js_contains`…) работят през таблица ALIAS.

**Скелетът на всеки JS урок:**

```javascript
{ id:"t1", type:"changed", value:"", err:"empty", weight:1000 },
{ id:"t2", type:"src_changed", err:"untouched", weight:950 },
{ id:"t3", type:"dom_has", value:'script[data-from="script.js"]', err:"not-linked", weight:850 },
{ id:"t4", type:"runs", err:"crashed", errEmpty:"nothing-written", errTimeout:"frozen", weight:800 },
```

Правило: празно > непипнат скелет > гърми ли > резултат > структура.
Показва се НАЙ-ТЕЖКАТА паднала, не първата.

⚠ **НЕ ползвай `balanced` в JS уроци.** Той брои HTML тагове и се обърква
от къдравите скоби на цикли и функции. Махнат от урок 26 нататък.
Счупеният синтаксис се хваща от `runs`.

### ⚠ СЛАБИ ПРОВЕРКИ — платени поуки

Преди да напишеш проверка: **може ли да падне?** Не може ли — украса.

Хванати на място:
- `src_not_contains: "* 1247"` при код `1247 * 3` — не съвпада с нищо
- `src_count: "if"` — брои и вътре в думи, и в `else if`
- `src_count` за `"=="` при решение с `===` — **броенето НЕ се припокрива**,
  три `===` дават ТРИ попадения, не шест
- `src_count` с `min:1, max:1` при решение с ДВЕ обявления

### ⚠ Как работи `norm()`
Свива интервалите до един, но НЕ ги маха.
Значи `" == "` НЕ съвпада с `"=="`, а `"if ("` НЕ съвпада с `"if("`.
Проверявай с реалния текст, не с предположение.

## 6. ⚠ ЗАДЪЛЖИТЕЛНО: как се проверява урок

Всеки урок носи поле `solution` до `slug`. Не е текст, не се превежда.

`/bg/lessontest` минава през целия курс и иска:
1. **Стартовият код ПАДА** — мине ли, урокът се минава без работа
2. **Решението МИНАВА** — падне ли, урокът е непроходим
3. Всяка грешка има `why`
4. Всяка видима проверка има `checkLabel`

**Не се пише следващият урок, докато страницата не е зелена.**

Дълъг код в `solution` и `starterFiles` се пише като масив от редове:

```javascript
solution: [
  'const a = 1;',
  'console.log(a);',
].join('\n'),
```

Иначе шаблонните низове с `${}` вътре чупят файла.

Куизовете: `quiz: true`, НЕ `type: "quiz"`.

Пиши стъпало само ако казва НОВО нещо.

**Стандартният `index.html` за JS урок** — не се променя между уроците:

## 14. ⚠ Проверени факти — не ги пипай без нов източник

- dockview: `theme={themeDark}` — обект, не низ. CSS към `.dv-shell`.
- `EditorGrid` иска `w-full h-full`, не `flex-1`.
- Monaco иска `path={име}`.
- `code.content` в Supabase е низ. Няколко файла → JSON низ, по първи знак `{`.
- Всички низове през next-intl, `src/messages/{bg,en}/practice.json`, плоски.
  Английският фолбек МЪЛЧИ — хардкоднат български не се забелязва месеци.
- Превюто на десктоп ОСТАВА в дървото, само скрито. Размонтирана рамка =
  скриптът не се изпълнява = конзолата мълчи.
- В `guard.js` вътре в шаблонния низ ВСЯКА наклонена черта е двойна.
  `\s` става `s` тихо. Затварящ таг за скрипт дори в коментар разсипва страницата.
- Заклещено превю НЕ може да се засече. Рамката дели нишката с React.
  Предаването е защитено, защото минава през Worker.
- `Number("  ")` дава **0**, не NaN. `Number("three")` дава NaN.
- `console.log(Bean Street)` → SyntaxError. `console.log(Bean)` → ReferenceError.
- Грешка „Unexpected non-whitespace character after JSON" при `npm run dev`
  почти винаги е повреден кеш: `Remove-Item -Recurse -Force .next`
