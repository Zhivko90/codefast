## 2. Как работим

### 2.1 Промени в код
До 3 места — блокове за замяна. Повече — искай целия файл, върни целия файл.
Не гадай какво има във файла. Поискай го.

### 2.2 Команди
Една команда на ред. **Пълни пътища. Винаги. Над всеки блок код.**
PowerShell, не bash.

Скоби в пътя чупят PowerShell. За `[locale]` и `[slug]` ползвай `-LiteralPath`.

⚠ `New-Item` НЯМА параметър `-LiteralPath`.
⚠ `[IO.Directory]::CreateDirectory` иска ПЪЛЕН път — не работи след `cd`.
⚠ `Get-Content` и `Add-Content` искат `-Encoding UTF8` от двете страни,
иначе кирилицата излиза развалена.

### Как изглежда всеки отговор с код

**Команда, после код. Винаги в тази последователност, за всеки файл.**

Нов файл:

    New-Item -Path D:\Proekti\codefast\src\data\courses\webproject\lessons\01-start.js -ItemType File -Force; code D:\Proekti\codefast\src\data\courses\webproject\lessons\01-start.js

Съществуващ файл:

    code D:\Proekti\codefast\src\core\checkProblem.js

Нов файл в папка със скоби:

    [IO.File]::WriteAllText('D:\Proekti\codefast\src\content\courses\webproject\bg\01-start.json',''); code D:\Proekti\codefast\src\content\courses\webproject\bg\01-start.json

Нова папка:

    New-Item -Path D:\Proekti\codefast\src\data\courses\webproject\lessons -ItemType Directory -Force

⚠ Никога не давай код без командата над него. Човекът не помни къде
живее файлът и не бива да го помни.

⚠ При урок се дават ЧЕТИРИ блока наведнъж, всеки със своята команда:
логика, BG текст, EN текст, `outline.js`. Накрая — регистърът.

### Как се тества

Не давай четири кода за лепване в едно съобщение. Давай ги ЕДИН ПО ЕДИН
и чакай резултата. Човекът тества, връща какво е излязло, после следващият.

Не карай човека да рови в терминала. Дай му `code файл.js`.

### 2.3 Отговорите
Първо докладвай КАКВО е счупено. После давай кода.
Три файла на отговор, максимум.
Не мисли на глас. Реши и дай готовото.

### 2.4 Коментари в кода
**Малко.** Само това, което обяснява РЕШЕНИЕ — защо е така, а не иначе.
Не описвай какво прави кодът. Не повтаряй паспорта в коментар.

## 5. Проверката

`src/core/checkProblem.js`. Получава ЕДИН сглобен низ HTML.

**Типове:** `dom_*`, `style_*`, `text_*`, `axe_clean`, `balanced`, `changed`,
`code_*`, и за изпълним код: `runs`, `returns`, `throws`, `logs`,
`src_contains`, `src_not_contains`, `src_count`, `src_changed`.

**Скелетът на всеки урок:**
{ id:"t1", type:"changed", value:"", err:"empty", weight:1000 }
{ id:"t2", type:"src_changed", err:"untouched", weight:950 }
{ id:"g1", type:"balanced", err:"not-closed", weight:900, guard:true }
{ id:"t3", type:"dom_has", value:'script[data-from="script.js"]', weight:850 }
{ id:"t4", type:"runs", err:"crashed", errEmpty:"...", errTimeout:"...", weight:800 }
Правило: празно > синтаксис > гърми ли > резултат > структура.
Показва се НАЙ-ТЕЖКАТА паднала, не първата.

⚠ **СЛАБИ ПРОВЕРКИ.** Преди да напишеш проверка, питай се
**може ли да падне**. Не може ли — това не е проверка, а украса.
Платена поука: `pattern: "."`, `src_contains: "function"`,
`src_not_contains: "* 1247"` при код, в който пише `1247 * 3`.

## 6. ⚠ ЗАДЪЛЖИТЕЛНО: как се проверява урок

Всеки урок носи поле `solution` до `slug`. Не е текст, не се превежда.

`/bg/lessontest` минава през целия курс и иска:
1. **Стартовият код ПАДА** — мине ли, урокът се минава без работа
2. **Решението МИНАВА** — падне ли, урокът е непроходим
3. Всяка грешка има `why`
4. Всяка видима проверка има `checkLabel`

**Не се пише следващият урок, докато страницата не е изцяло зелена.**

⚠ Страницата е закована за `js`. За този курс смени `const COURSE`
в `src/app/[locale]/lessontest/page.js`.

⚠ **ЗА ПРОЕКТЕН КУРС ТРЯБВА ВЕРИЖНА ПРОВЕРКА** — не съществува още.
Всеки урок тръгва от работата на ученика, не от постоянен стартов код.
Значи тестът минава курса като симулиран ученик: решението на урок N
става старт на урок N+1. Хваща счупени вериги — решение на урок 7,
което трие нещо, което урок 19 очаква.
**Построй я, преди да пишеш урок 3.**

## 7. Форматът на урок

```javascript
export default {
  id: 1,
  type: "web",
  label: "coding",
  entry: "index.html",
  starterFiles: { "index.html": `...`, "style.css": `...`, "script.js": `...` },
  checks: [ ... ],
  solution: `...`,
  blocks: [ ... ],
  slug: "01-..."
};
```

Текстът е в `content/courses/webproject/{bg,en}/01-....json`,
плоски ключове с точка: `checkLabels.t1`, `why.no-return`,
`hint2.no-return`, `blocks.0.text`, `blocks.3.items.0`.

Куизовете ползват `quiz: true`, НЕ `type: "quiz"`.

**Стълбата от подсказки:**
why.X    Симптомът. Показва се сам.
hint2.X  Къде да гледаш.
hint3.X  Разработен пример. ПОДОБЕН случай, не неговия.
hint4.X  Какво да направиш. НЕ готовият код.
Пиши стъпало само ако казва НОВО нещо.

**Проектни уроци:** `project: true`. `saveProject`/`fetchProject`
пренасят кода от урок в урок. Пази се ПОСЛЕДНОТО МИНАЛО състояние,
не текущото — значи счупен урок не убива следващите.

## 14. ⚠ Проверени факти — не ги пипай без нов източник

- `New-Item` няма `-LiteralPath`. Скоби в пътя → `[IO.Directory]::CreateDirectory`
  с ПЪЛЕН път.
- dockview: темата е `theme={themeDark}` — обект, не низ.
- dockview CSS се насочва към `.dv-shell`, не към `.dockview-theme-dark`.
- `EditorGrid` иска `w-full h-full`, не `flex-1` — иначе височината е нула.
- Monaco иска `path={име}` — иначе undo историята се смесва.
- `code.content` в Supabase е низ. Няколко файла → JSON низ, разпознава се
  по първия знак `{`.
- Всички низове минават през next-intl, ключове в
  `src/messages/{bg,en}/practice.json`, плоски. Английският фолбек МЪЛЧИ —
  хардкоднат български не се забелязва месеци.
- Превюто на десктоп трябва да ОСТАВА в дървото, само скрито.
  Размонтирана рамка = скриптът не се изпълнява = конзолата мълчи.
- В `guard.js` вътре в шаблонния низ ВСЯКА наклонена черта е двойна.
  `\s` става `s` и образецът умира тихо. Затварящ таг за скрипт дори
  в коментар разсипва страницата.
- Заклещено превю НЕ може да се засече. Рамката дели нишката с React.
  Предаването е защитено, защото минава през Worker.
